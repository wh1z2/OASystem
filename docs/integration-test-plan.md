## 2. 测试范围定义

### 2.1 测试功能模块

| 模块 | 优先级 | 测试重点 |
|------|--------|----------|
| 用户认证模块 | P0 | 登录/登出/Token刷新/权限获取 |
| 审批工单管理 | P0 | CRUD/状态流转/列表查询 |
| 审批流程控制 | P0 | 提交/审批/拒绝/撤销/重新编辑 |
| 审批历史记录 | P1 | 查询/代审批标识 |
| 权限控制 | P1 | 多级权限/代审批逻辑 |

### 2.2 接口清单（14个核心接口）

**认证接口 (2个)**
- `POST /auth/login` - 用户登录
- `GET /auth/info` - 获取当前用户信息

**工单CRUD接口 (5个)**
- `POST /approvals` - 创建工单
- `PUT /approvals/{id}` - 更新工单
- `DELETE /approvals/{id}` - 删除工单
- `GET /approvals/{id}` - 查询详情
- `GET /approvals` - 列表查询（分页）

**状态流转接口 (5个)**
- `POST /approvals/{id}/submit` - 提交工单
- `POST /approvals/{id}/approve` - 审批通过
- `POST /approvals/{id}/reject` - 审批拒绝
- `POST /approvals/{id}/reedit` - 重新编辑
- `POST /approvals/{id}/revoke` - 撤销申请

**列表查询接口 (3个)**
- `GET /approvals/todo` - 待办列表
- `GET /approvals/done` - 已办列表
- `GET /approvals/my` - 我的申请
- `GET /approvals/{id}/history` - 审批历史

## 4. 测试执行步骤

### 4.1 预检查阶段

```
□ 1. 确认MySQL服务已启动
□ 2. 执行 database/init.sql 初始化数据
□ 3. 启动后端服务 (mvn spring-boot:run)
   - 检查控制台无报错
   - 确认端口8080被占用
□ 4. 启动前端服务 (npm run dev)
   - 确认端口3000
□ 5. 浏览器访问 http://localhost:8080/api 确认后端可访问
□ 6. 浏览器访问 http://localhost:3000 确认前端可访问
```

### 4.2 执行顺序

```
Phase 1: 认证模块测试
  └─ TC-LOGIN-001 ~ TC-LOGIN-003
  └─ TC-LOGIN-ERR-001 ~ TC-LOGIN-ERR-004
  └─ TC-SEC-001

Phase 2: 工单CRUD测试
  └─ 使用admin账号：创建、查询、更新、删除工单
  └─ 验证分页功能

Phase 3: 状态流转测试
  └─ TC-FLOW-001 (正常流程)
  └─ TC-FLOW-002 (拒绝流程)
  └─ TC-FLOW-004 (撤销流程)

Phase 4: 权限测试
  └─ TC-FLOW-003 (代审批)
  └─ TC-SEC-002 (越权)

Phase 5: 边界测试
  └─ TC-SEC-003 (XSS)
```

## 8. 风险评估与应对措施

### 8.1 风险清单

| 风险ID | 风险描述 | 可能性 | 影响 | 应对措施 |
|--------|----------|--------|------|----------|
| R1 | 后端服务启动失败 | 中 | 高 | 检查maven依赖、检查8080端口占用、查看日志 |
| R2 | 数据库连接失败 | 中 | 高 | 确认MySQL服务状态、检查application-dev.yml配置 |
| R3 | 前端代理配置错误 | 高 | 中 | 检查vite.config.js，确认无rewrite规则 |
| R4 | Token过期机制异常 | 低 | 中 | 使用测试工具生成短时效token验证 |
| R5 | 跨域问题 | 低 | 中 | 确认后端CORS配置和前端代理配置 |

### 8.2 应急预案

**场景1: 后端无法启动**
```bash
# 1. 检查端口占用
netstat -ano | findstr 8080

# 2. 清理重新编译
cd oa-system-main
mvn clean compile

# 3. 查看详细错误日志
mvn spring-boot:run -X
```

**场景2: 数据库连接失败**
```bash
# 1. 测试MySQL连接
mysql -u root -p -e "SELECT 1"

# 2. 重新初始化数据库
mysql -u root -p < database/init.sql
```

**场景3: 前端代理不生效**
```bash
# 1. 确认vite.config.js配置正确
# 2. 必须重启前端服务
npm run dev

# 3. 检查代理是否生效（浏览器DevTools Network标签页查看请求URL）
```
