# 前端交互测试计划 —— 审批规则与默认审批人模块

## 1. 测试概述

### 1.1 测试范围
本测试计划覆盖**审批规则配置**和**默认审批人自动解析**功能在前端的交互表现。

### 1.2 测试目标
- 验证审批规则配置页面的增删改查交互正常
- 验证工单创建/提交时审批人自动解析的交互表现
- 验证权限控制（页面访问、按钮显隐）正确
- 验证异常场景下的前端反馈（错误提示、兜底处理）

### 1.3 测试环境
| 项目 | 配置 |
|------|------|
| 浏览器 | Chrome 120+ / Edge 120+ |
| 前端地址 | http://localhost:3000 |
| 测试账号 | admin/admin123, manager/manager123, user/user123 |
| 视口 | 1920×1080（桌面端为主，移动端抽查） |

## 4. 已知问题与注意事项

1. **权限问题**：manager 角色默认无 `role_manage` 权限，无法访问 `/approver-rules`，需使用 admin 账号测试该页面。
2. **自动解析兜底**：创建草稿时解析失败不会阻断，但提交时会严格校验并给出明确提示。
3. **浏览器兼容性**：表单设计器使用了拖拽 API，建议在 Chrome/Edge 下测试。
