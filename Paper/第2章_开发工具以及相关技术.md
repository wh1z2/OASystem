# 第2章 开发工具以及相关技术

本章对工单流程自动化系统在设计与实现过程中所使用的开发工具和相关技术进行系统说明。本章的写作思路是先介绍项目所采用的集成开发环境、构建工具、数据库管理工具及接口测试工具,再分别从前端、后端、数据库三个维度阐述系统所依托的核心技术,并结合本毕业设计的实际场景说明每项技术的选型理由。所列工具与技术版本均与项目实际依赖文件(`package.json`、`pom.xml`、`application.yml`、`vite.config.js` 等)中记录的内容保持一致。

## 2.1 开发工具

软件开发工具是软件工程实践的基础设施,直接影响开发效率、调试体验与代码质量。本节按"前端、后端、数据库、接口测试"四个维度,介绍本项目使用的全部开发工具。

### 2.1.1 前端开发工具

本项目前端的代码编辑、调试与构建均在 Visual Studio Code(以下简称 VS Code)中完成。VS Code 是 Microsoft 推出的开源、跨平台的轻量级源代码编辑器,以丰富的插件市场、出色的智能提示能力以及对 Git 的原生集成,成为前端工程领域使用最为广泛的开发工具之一。在本项目中,VS Code 配合 Volar 插件为 `.vue` 单文件组件提供语法高亮与类型推断,配合 ESLint、Prettier 插件统一代码风格,并通过内置终端面板执行 `npm run dev`、`npm run build` 等命令。选用 VS Code 而非 WebStorm,主要考虑其启动速度快、占用内存小,且开源生态对 Vue 3、Vite、Tailwind CSS 等新兴技术的插件支持更为及时。

在前端构建工具方面,本项目采用 npm 作为包管理工具,以 Vite 5.2.0 作为构建与开发服务器。npm 通过 `package.json` 完成依赖安装与脚本调度,本项目以 `dependencies/devDependencies` 区分运行时与开发时依赖,并通过 `scripts` 字段统一定义 `dev`、`build`、`preview` 三条命令。Vite 由 Vue.js 作者尤雨溪主导开发,基于浏览器原生 ES Modules 实现按需编译,开发模式下可达到毫秒级冷启动与热更新;生产构建阶段则封装 Rollup 完成代码打包、Tree-Shaking 与压缩优化。本项目通过 `vite.config.js` 配置 `@/` 路径别名简化模块引用,并通过 `server.proxy` 将 `/api` 前缀请求代理至本地 8080 端口的后端服务,以此在前后端分离开发模式下规避跨域问题。

### 2.1.2 后端开发工具

本项目后端开发主要在 IntelliJ IDEA(以下简称 IDEA)中完成。IDEA 由 JetBrains 公司开发,是 Java 后端领域公认的主流 IDE,对 Spring Boot、MyBatis、Maven 等框架提供一流的内置支持,能够自动识别 `@Component`、`@Service`、`@RestController` 等注解并以图形化方式展示 Bean 的依赖关系,且具备强大的重构、调试与数据库可视化能力。在本项目中,IDEA 主要用于运行主类 `OaSystemApplication`、调试 COLA StateMachine 状态流转、查看 MyBatis-Plus 自动生成的 SQL 以及对 JaCoCo 覆盖率报告进行可视化分析。选用 IDEA 而非 Eclipse 等同类工具,主要因其对 Spring 生态的开箱即用支持以及更优秀的代码智能提示与重构体验,能够显著提升中等规模 Java 工程的开发效率。

在后端构建工具方面,本项目采用 Apache Maven。Maven 基于"约定优于配置"理念,通过 `pom.xml` 以 XML 形式声明项目坐标、依赖与构建插件。本项目通过继承 `spring-boot-starter-parent` 3.2.3 获得官方依赖版本管理,通过 `<properties>` 节点统一维护 MyBatis-Plus、COLA StateMachine、JJWT 等关键依赖版本号,并通过 `spring-boot-maven-plugin` 实现一键打包为可独立运行的 fat-jar、通过 `jacoco-maven-plugin` 在测试阶段产出代码覆盖率报告。选用 Maven 而非 Gradle,主要因其在 Spring 生态中文档更成熟、示例更丰富,适合本科毕业设计这一中等规模工程。

### 2.1.3 数据库工具

本项目在数据库管理方面主要使用 MySQL Workbench 8.0 与 IDEA 自带的 Database 插件协同完成。MySQL Workbench 是 MySQL 官方推出的可视化设计与管理工具,在本项目中承担 E-R 图绘制、迁移脚本(如 `database/refresh-token-migration.sql`)的执行以及连接数与慢查询的监控等任务。IDEA 自带的 Database 插件则提供智能 SQL 补全、表/列名联想以及"代码—SQL 编辑—数据预览"三视图切换的能力,是日常开发中使用频率更高的工具,在调试 MyBatis-Plus 自动生成的 SQL 时尤为高效。

总体而言,数据库可视化工具在本项目中的核心作用在于:降低 SQL 使用门槛、辅助完成开发库与测试库的数据同步,以及通过 EXPLAIN 执行计划帮助识别索引使用情况,从源头规避审批列表分页、待办任务聚合等热点接口上的性能瓶颈。

### 2.1.4 接口测试工具

本项目在前后端联调阶段使用 Apifox 作为接口调试与测试工具。Apifox 是国内推出的"API 设计、调试、文档、Mock、自动化测试"一体化平台,可视为 Postman、Swagger 与 JMeter 的功能集合体。在本项目中,Apifox 主要承担三方面工作:其一,作为接口契约的中心化存储,后端按模块录入 Spring Boot Controller 暴露的全部接口(如登录、Token 刷新、审批查询与审批操作等),前端据此完成请求封装;其二,作为请求调试工具,通过环境变量与预/后置脚本自动注入 `Authorization` 鉴权头,简化带鉴权接口的反复调试;其三,作为自动化测试工具,可将"创建审批—提交—审批通过—驳回—重新编辑"等多接口编排为用例集,辅助验证 COLA StateMachine 状态流转逻辑的正确性。

选用 Apifox 而非 Postman,主要因其对中文开发者的本地化支持更友好,且将 Mock 数据生成与接口调试紧密整合;在后端尚未实现某些接口的阶段,前端可直接通过 Apifox 的 Mock 服务获取符合契约的模拟数据,显著降低前后端开发的耦合度。

## 2.2 开发技术

本节对系统所采用的核心开发技术进行说明,涵盖前端、后端、数据库三个层面。

### 2.2.1 前端技术

#### (1) Vue 3 渐进式前端框架

Vue.js 是由尤雨溪团队维护的渐进式前端框架,核心库专注于视图层,具备易上手、生态完善等特点。本项目基于 Vue 3.4.21 实现,全面采用 Composition API 与 `<script setup>` 语法糖。Composition API 以"组合函数"形式组织组件逻辑,有效解决 Options API 在大型项目中"逻辑分散、难以复用"的问题;`<script setup>` 则使变量、函数等顶层声明可直接在模板中使用,代码更紧凑直观。Vue 3 的核心特性包括:基于 ES6 `Proxy` 重写的响应式系统,可代理整个对象并支持 Map/Set 等容器类型;经过优化的虚拟 DOM 与 Diff 算法;贯穿框架的组件化思想,通过 `props`、`emit`、`provide/inject` 实现多层次通信;以及良好的 TypeScript 友好性。本项目中,审批管理(`ApprovalManage.vue`)、审批详情、表单设计器等复杂页面均按"页面级组件 + 子组件"模式拆分,符合"单一职责、单文件不超过 300 行"的工程规范。

#### (2) Vue Router 与 Pinia

Vue Router 4.3.0 是 Vue.js 官方维护的客户端路由方案,本项目通过 `src/router/index.js` 集中定义全部路由,并在全局前置守卫中实现 JWT Token 校验与权限路由分发,与后端 Spring Security 的"接口级权限控制"形成纵深防御。Pinia 2.1.7 是 Vue 官方推荐的下一代状态管理库,相比 Vuex 取消了 mutation 中间层、API 更简洁、对 Composition API 与 TypeScript 支持更佳。本项目以"领域"为粒度组织 store——`stores/auth.js` 管理双 Token 状态,`stores/approval.js` 管理审批列表,`stores/user.js` 管理 RBAC 数据,以避免单一 store 臃肿与模块状态污染。

#### (3) Tailwind CSS 与 UI 设计体系

Tailwind CSS 3.4.3 是"实用优先"(utility-first)的 CSS 框架,提供大量原子化工具类,开发者通过组合工具类直接表达样式。本项目通过 `tailwind.config.js` 自定义 primary/success/warning/danger 四套色阶,并在 `src/assets` 中通过 `@apply` 封装 `card`、`btn-primary` 等通用样式,实现"原子类组合"与"组件类抽象"的平衡。选用 Tailwind 而非 Element Plus、Ant Design Vue 等成品组件库,主要因其在 UI 原创性与产物体积(经 PurgeCSS 后通常仅十几 KB)上更具优势,适合本毕业设计对视觉表现的展示需求。配合 PostCSS 8.4.38 与 Autoprefixer 10.4.19,Tailwind 可自动完成 CSS 浏览器兼容降级。

#### (4) Axios 网络请求库与 Heroicons 图标库

Axios 1.6.8 是基于 Promise 的 HTTP 客户端,提供统一的请求/响应拦截器、请求取消与自动 JSON 转换等能力。本项目在 `src/api/` 下基于 Axios 封装统一的请求实例:请求拦截器自动注入 `Authorization` 鉴权头;响应拦截器识别 401 状态码并触发 Refresh Token 续期,实现 Token 失效后的无感刷新;同时统一捕获网络与业务异常并通过全局通知组件呈现。@heroicons/vue 2.1.1 是 Tailwind Labs 推出的开源 SVG 图标库 Vue 适配版,与 Tailwind 在视觉风格上高度一致,在导航栏、按钮与状态徽章等位置广泛使用。

### 2.2.2 后端技术

#### (1) Spring Boot 3.2.3 主框架

Spring Boot 是 Spring 团队推出的"约定优于配置"快速开发框架,通过 Starter 依赖、自动装配与内嵌 Servlet 容器三大特性大幅简化 Spring 应用搭建。本项目基于 Spring Boot 3.2.3、运行环境 Java 17(LTS);Spring Boot 3.x 已全面将基础包从 `javax.*` 迁移至 `jakarta.*`,以适配 Jakarta EE 9 标准。其核心特性在本项目中体现为:IoC 与 DI 通过 `@Service`、`@RestController` 等注解实现各层解耦;AOP 用于日志、权限、事务等横切关注点的统一处理;自动装配通过 `spring-boot-starter-web/validation/security` 一站式引入 Tomcat、Hibernate Validator 与安全过滤器链;基于 `application.yml` 与 `spring.profiles.active` 的外部化配置实现多环境无缝切换。

#### (2) Spring Security 与 JWT 鉴权

本项目结合 Spring Security 的过滤器链与 JWT(JSON Web Token)实现无状态鉴权:用户登录后,后端返回 Access Token(30 分钟)与 Refresh Token(7 天)一对凭证;Access Token 用于日常接口调用,过期后客户端通过 Refresh Token 调用 `/api/auth/refresh` 自动续签,在易用性与安全性之间取得平衡。JWT 实现依赖 jjwt 0.12.3,采用 `jjwt-api/jjwt-impl/jjwt-jackson` API 与实现分离的设计;签名密钥与过期时间在 `application.yml` 中通过 `jwt.secret`、`jwt.expiration`、`jwt.refresh-expiration` 集中管理。

#### (3) MyBatis-Plus ORM 框架

MyBatis-Plus 3.5.10.1 是国内广泛使用的 MyBatis 增强工具,在不改变 MyBatis 原有特性的前提下提供 `BaseMapper` 通用 CRUD、`LambdaQueryWrapper` 类型安全条件构造器以及 `PaginationInnerInterceptor` 分页插件等开箱即用能力,本项目额外引入 `mybatis-plus-jsqlparser` 模块以支持自 3.5.9 起拆分出的 JSqlParser 能力。ORM 框架的价值在于将面向对象的领域模型与关系数据库映射,免除手写大量样板 SQL。本项目中,审批工单、用户、角色、表单模板等核心实体均通过 `@TableName`、`@TableField`、`@TableId` 注解完成映射,Service 层直接调用 `getById`、`page` 等方法,既提升开发效率,又满足"禁止在 Service 中编写 SQL 拼接"的工程规范。

#### (4) Alibaba COLA StateMachine 状态机引擎

工单审批的核心是状态的有序流转,选用合适的状态机引擎是后端架构的关键决策。本项目采用阿里巴巴开源的 COLA StateMachine 5.0.0 实现审批工作流:其链式 DSL 用于定义状态、事件、转换规则、守卫条件(Condition)与转换动作(Action),无状态、内存级的设计使状态机实例可作为 Spring Bean 单例存在,具备良好吞吐性能。本项目审批工单依据 `DRAFT → PROCESSING → APPROVED/RETURNED → DRAFT` 与 `PROCESSING → REVOKED` 等规则流转,COLA 的 DSL 模型与业务语义高度吻合,后续若新增"会签""或签"等高级流程,只需在状态机定义中追加规则,符合开放-封闭原则。

#### (5) 辅助技术

Lombok 通过编译期注解自动生成 getter/setter、`equals`、`hashCode` 等样板代码,在实体类与 DTO 中显著减少冗余;FastJSON2 2.0.46 用于解析与序列化表单设计器中的动态 `form_data` JSON 字段;测试方面通过 `spring-boot-starter-test` 引入 JUnit、Mockito、AssertJ,通过 `spring-security-test` 模拟带鉴权请求,通过 JaCoCo 0.8.11 在 `mvn test` 阶段产出 HTML 覆盖率报告,作为工程质量的量化证据。

### 2.2.3 数据库技术

本项目采用关系型数据库 MySQL 8.0 作为持久化层。MySQL 由 Oracle 公司维护,是开源关系型数据库中市场占有率最高的产品之一,具备规范化的数据模型、完整的 ACID 事务、标准 SQL 与成熟的索引机制等优势。MySQL 8.0 相比 5.7 引入了原子化 DDL、窗口函数与 CTE、默认 `utf8mb4` 字符集以及 Redo/Undo Log 并发优化等关键特性。

本项目数据库主要包含用户、角色、用户-角色关联、表单模板、审批工单、审批日志、Refresh Token 等表。此类数据结构化特征强、实体关联明确,且需频繁执行多表联合查询(如某用户在某状态下的全部待办)与事务性更新(如审批状态变更需与日志写入保持原子性),因此关系型数据库是合理选择。选用 MySQL 而非 PostgreSQL、Oracle 等其他关系型数据库,理由有三:其一,开源免费,适合本科毕业设计的成本约束;其二,与 Spring Boot、MyBatis-Plus 的生态整合最为成熟,`mysql-connector-j` 由 Oracle 官方维护并已被 Spring Boot 父 POM 自动管理版本;其三,在国内互联网行业部署最为广泛,有助于工程经验向工业实践迁移。本项目未引入 MongoDB、Redis 等非关系型数据库,主要因审批业务对一致性的要求高于吞吐量,关系型数据库的强一致性事务模型已能覆盖现阶段全部业务场景。

## 本章小结

本章围绕本毕业设计所使用的开发工具与相关技术进行了系统阐述。在开发工具层面,前端采用 VS Code 配合 npm 与 Vite 完成代码编辑与构建,后端采用 IntelliJ IDEA 配合 Maven 完成编码与构建,数据库使用 MySQL Workbench 与 IDEA Database 插件进行可视化管理,接口测试与联调使用 Apifox。在开发技术层面,前端基于 Vue 3 + Vue Router + Pinia + Tailwind CSS + Axios 构成完整的单页应用技术栈,后端基于 Spring Boot 3.2.3 + Spring Security + MyBatis-Plus + COLA StateMachine + JJWT 构建分层清晰的工作流后端,数据库采用 MySQL 8.0 提供事务性持久化能力。上述工具与技术的组合,在易用性、性能、可维护性以及生态成熟度上均取得了较好的平衡,为后续章节中可行性分析、需求分析、总体设计与详细设计的展开奠定了技术基础。
