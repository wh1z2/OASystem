# 6 系统测试

## 6.1 测试背景与目的

系统测试是软件开发生命周期中至关重要的质量保证环节，通常在单元测试和集成测试完成后执行，旨在验证整个软件系统是否满足规定的需求规格。对于本工单审批自动化系统而言，由于涉及状态机驱动的审批流转、基于角色的权限控制、动态表单渲染以及多级审批规则解析等复杂业务逻辑，全面的系统功能测试尤为必要。测试的目的在于验证各功能模块是否按第五章的实现逻辑正常运行，及时发现并修正缺陷，确保系统在上线后具备足够的稳定性、可用性和数据一致性。

本系统的测试环境沿用第五章所述的开发环境：后端运行于 Windows 11 操作系统，采用 Spring Boot 3.2.3 框架、Java 17 运行时及 MySQL 8.0 数据库；前端基于 Vue 3.4.21 与 Vite 5.2.0 构建工具，通过 Chrome 浏览器进行功能验证。测试方法采用黑盒功能测试，按照模块设计测试用例，覆盖各核心业务路径的正常流程与关键边界条件。
